## How to Contribute

### 1. Fork the Repository

Start by forking the MediaSort repository to your GitHub account. This will create a copy of the project that you can freely modify.

### 2. Clone Your Fork

Clone your fork to your local machine:

```bash
git clone https://github.com/your-username/MediaSort.git
cd MediaSort
```

### 3. Create a Branch

Create a new branch for your modifications:

```bash
git checkout -b your-branch-name
```

### 4. Make Your Changes

Make the desired changes to the code. Be sure to follow the project's coding conventions.

### 5. Test Your Changes

Ensure that your changes work correctly and don't introduce any new bugs.

### 6. Commit Your Changes

Commit your changes with a clear and descriptive message:

```bash
git add .
git commit -m "Concise description of your changes"
```

### 7. Push Your Changes

Push your changes to your GitHub fork:

```bash
git push origin your-branch-name
```

### 8. Create a Pull Request

Go to the original project's GitHub page and create a new Pull Request from your branch.

## Contribution Guidelines

- Ensure your code adheres to the project's style standards.
- Document all new features or significant modifications.
- Add tests for new features if possible.
- Update documentation if necessary.

## Reporting Bugs

If you find a bug, please add it to the issues section on GitHub with as much detail as possible on how to reproduce it.

## Suggesting Improvements

Improvement suggestions are always welcome. Open an issue on GitHub to discuss your ideas before you start working on them.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and collaborative environment. Any inappropriate behavior will not be tolerated.

Thank you for contributing to MediaSort!
